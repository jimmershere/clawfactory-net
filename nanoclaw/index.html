The Platforms

Different architectures. Same mission.

OpenClaw and NanoClaw both exist to give you a real AI assistant that does things. They just take different roads to get there.

NanoClaw Container-first

Built by Gavriel Cohen in January 2026. ~3,900 lines of code across 15 files. Every agent session runs in its own Docker or Apple Container — OS-level isolation by default. Karpathy called it "manageable, auditable, flexible." 24k+ GitHub stars and growing fast.

  • OS-level container isolation — agents can't touch what you don't mount
  • Tiny, readable codebase — understand it in under an hour
  • Built on Anthropic's Claude Agent SDK natively
  • Per-group memory isolation
  • Docker + Apple Container support
  • Agent swarms — first personal AI to support this
🦞
OpenClaw Ecosystem-first

The original personal AI agent platform. 5,400+ skills, 2.1M+ downloads, deployed by Baidu and Alibaba at enterprise scale. Rich channel support, cron scheduling, tiered memory, and the most mature skill ecosystem in the space. The platform with the biggest surface area.

  • 5,400+ skills — largest ecosystem by far
  • Multi-channel: Telegram, Discord, WhatsApp, Signal, Slack
  • Cron scheduling, heartbeat monitoring, workspace memory
  • Enterprise-validated by Baidu and Alibaba
  • ClawHub skill marketplace + active community
  • Sub-agents, ACP harness, coding agents
🎵

Ebony and Ivory

The "claw wars" narrative is a media creation. Two great tools built by different people for different philosophies — container security vs ecosystem breadth — aren't enemies. They're options. ClawFactory exists to support builders on both, vet the work that comes from both, and let the community decide what wins on merit.
