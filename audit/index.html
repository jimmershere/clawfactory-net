What does this mean for you? If you've installed OpenClaw community skills without vetting them, there's roughly a 1-in-6 chance at least one of them contains a pattern our scanner flags as HIGH risk. That includes patterns like remote code execution, reverse shells, and unauthorized filesystem access.

This isn't about malicious intent — many of these are legitimate tools that happen to use powerful system calls. But unvetted ≠ safe. You deserve to know what's running on your machine.

Findings

Top Vulnerability Patterns Found

These are the most common risk patterns across all 199 scanned skills, ranked by frequency. Each one is a legitimate security concern when present in an unvetted AI agent skill.

pip install — installs packages 113 occurrences · HIGH context

Skills that run pip install can install any Python package onto your system — including malicious ones. Without sandboxing, this runs directly on your host machine.

✓ ClawFactory fix: Gate 2 sandbox isolates all package installs inside containers. Nothing touches your host.

curl pipe to shell — remote code execution 42 occurrences · CRITICAL

curl https://... | bash downloads code from the internet and executes it immediately. No review. No verification. Whatever is at that URL runs on your machine.

✓ ClawFactory fix: Gate 1 blocks this pattern outright. It never reaches the registry.

Python subprocess — shell execution 58 occurrences · HIGH context

Python's subprocess module can execute arbitrary shell commands. Legitimate in many tools — but requires human review to verify intent and scope.

✓ ClawFactory fix: Gate 3 human review evaluates intent. Subprocess ≠ automatic block, but it always gets eyes on it.

Reading environment variables 43 occurrences · MEDIUM

Skills that read process.env can access your API keys, tokens, and credentials stored as environment variables — including your Anthropic key.

✓ ClawFactory fix: Flagged for review. Skills with legitimate env var needs are documented; others are blocked.

Global npm install 36 occurrences · MEDIUM

npm install -g installs packages system-wide, modifying your global Node.js environment. Can introduce malicious code that persists across sessions.

✓ ClawFactory fix: Sandboxed execution prevents global installs from affecting the host.

How We Did This

Methodology

This audit used ClawFactory's Gate 1 static analysis pipeline — the same scanner that runs on every skill submitted to our registry.

01
Source
GitHub API — all public repositories tagged with the openclaw-skill topic, sorted by stars. 416 repos found, 199 scanned before rate limits.
02
Scanner
Gate 1 static pattern analysis — 12 HIGH-risk patterns and 13 MEDIUM-risk patterns matched against all .md, .sh, .js, .ts, .py, and .json files in each repo.
03
Limitations
Static analysis only — no sandboxed execution (Gate 2) or human review (Gate 3). Some flags are false positives. HIGH rating = pattern match, not confirmed malicious intent.
